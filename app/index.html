Implementation Summary

Data model

To mock the data needed, a couple of objects were created which can be found under the /data folder.

Record basically means one event, something that happens during the course of a frame. It can be one of several categories, eg. Scripting or Painting. In reality each Record could have several records itself, but for simplicity reasons this wasn't implemented

Frame is basically a collection of Records.

And finally, there is an object which initializes the frames array and emulates an asynchronous call, as it would happen if we fetched them from a database.

D3 and Graphs

There is one many D3 object, called timeline. This object contains 2 groups, one for frames and one for records. Also, there is a brush which allows you to zoom in for the frames and pan, in order to see the records you want. Finally, by hovering on the records, you can see more details.

Reach components

There is a <Timeline> component to host the whole graph. It didn't make any sense to create any more, based on the requirements. The only thing that could be another component is the tooltip, but only for the markup. The functioanlity wouldn't be encapsulated in the component

Implementation duration

It took me around 16-20 hours, including the time to get more familiar with D3
